SCHEMA COLONNE FOGLIO "Candidature"
====================================

Crea un Google Sheet con UNA tab chiamata "Candidature" (esatta, maiuscola C).
Nella riga 1 incolla queste intestazioni (una per cella, da A1 ad AV1).

Suggerimento: copia tutta la riga sotto (sono separate da TAB) e incollala in A1:

timestamp	id	nome	cognome	email	telefono	citta	posizione	q1	q2	q3	q4	q5	q6	q7	q8	q9	q10	q11	q12	q13	q14	q15	q16	relazionale	mindset	organizzazione	flessibilita	fitSettore	potenziale	overall	categoria	q1_time	q2_time	q3_time	q4_time	q5_time	q6_time	q7_time	q8_time	q9_time	q10_time	q11_time	q12_time	q13_time	q14_time	q15_time	q16_time

Descrizione colonne:
- timestamp   : ISO 8601 (es. 2026-05-12T11:24:00Z)
- id          : ID univoco generato (es. c_a1b2c3)
- nome, cognome, email, telefono, citta, posizione : dati contatto
- q1..q16     : risposte (valori: si | incerto | no | na)
- 5 dimensioni 0-100 (relazionale, mindset, organizzazione, flessibilita, fitSettore)
- potenziale  0-100 (media delle prime 4 dimensioni)
- overall     0-100 (media di tutte e 5)
- categoria   : pronti | diamante | medio | non_ora
- q1_time..q16_time : tempo di risposta per domanda in millisecondi (vuoto se non disponibile)

Nota retrocompatibilità:
- Le 16 colonne q*_time sono in coda apposta: i fogli già esistenti senza queste colonne
  continuano a funzionare. Dopo l'aggiornamento del codice basta eseguire una volta
  ensureHeaders() dall'editor Apps Script per scrivere le nuove intestazioni; le righe
  esistenti restano com'erano e per i vecchi candidati il tempo risulterà non disponibile.

Formattazione consigliata (opzionale, manuale dal foglio):
- Riga 1: grassetto, blocco riga
- Colonna A (timestamp): formato data
- Colonne con q1..q16: testo
- Colonne dimensioni e overall: numero intero
- Colonne q*_time: numero intero (millisecondi)
- Filtro su intera riga 1 (Dati → Crea filtro)


SCHEMA SECONDA TAB "Posizioni"
==============================

Le posizioni lavorative gestite dalla dashboard admin vivono in una seconda tab
chiamata "Posizioni" (esatta). Viene creata in automatico al primo utilizzo, ma
se preferisci puoi crearla a mano con queste intestazioni in A1:

id	attivo	titolo	descrizione	ral	location	ordine	updated_at

Descrizione colonne:
- id          : slug univoco (es. qhse, elettricista; per nuove generate p_xxxxxxxx)
- attivo      : TRUE / FALSE — solo le attive vengono mostrate sul sito
- titolo      : nome della posizione mostrato sulla card
- descrizione : testo della card (1-2 righe)
- ral         : valore mostrato in basso a destra. Se contiene cifre il label
                visualizzato è "RAL", altrimenti "Contratto" (utile per "Da valutare")
- location    : badge in alto a destra (es. Napoli, Italia)
- ordine      : intero per ordinare le card crescente (10, 20, 30…)
- updated_at  : ISO 8601, scritto automaticamente ad ogni save

Bootstrap iniziale:
1. Apri Apps Script editor.
2. Esegui una volta la funzione seedPositions() — popola le 4 posizioni
   correnti (qhse, gare, elettricista, spontanea) con i valori del sito attuale.
   La funzione è idempotente: se già presenti aggiorna; altrimenti inserisce.
